Update agentbox's own .devcontainer setup to include Codex CLI, dogfooding the same pattern we generate for users.

## Scope

### `.devcontainer/Dockerfile` (custom stage)
- Add `RUN npm install -g @openai/codex` in the custom stage

### `.devcontainer/devcontainer.json`
- Add `"openai.chatgpt"` to VS Code extensions array
- Add `"source=agentbox-codex-config,target=/home/node/.codex,type=volume"` to mounts
- Add `containerEnv` with `"OPENAI_API_KEY": "${localEnv:OPENAI_API_KEY}"`
- Update `postStartCommand`:
  - Add `/home/node/.codex` to the chown command
  - Add Codex settings sync step (or create a manual codex-config.toml and sync script in our own .devcontainer)

### Codex settings for our own container
- Create `.devcontainer/codex-config.toml` with appropriate settings (full-auto + danger-full-access sandbox, since the container is externally sandboxed)
- Create `.devcontainer/sync-codex-settings.sh` or reuse the generated pattern

### Note
This is independent of all other beans — can be done in parallel. No Go code changes, just devcontainer config files.

## Implementation Plan

### Approach

Use `agentbox update --dir .` to regenerate all agentbox-managed files from the current templates, which already include Codex support. This is the cleanest dogfooding approach: the project's own `.devcontainer/` is generated by agentbox itself, so updating it should use the tool's own update command. This validates that the update flow works correctly and produces the right output.

Verified by test run: `agentbox update --dir /workspace` succeeds and produces exactly the expected diff (see Steps below). The agentbox-managed stage, firewall scripts, devcontainer.json, README, and dynamic-domains.conf are all regenerated with Codex support. Two new files are created. The custom stage (with `beans` tool install and env var) is preserved verbatim.

### Files to Create

- `.devcontainer/codex-config.toml` — Codex CLI configuration (approval_policy = "unless-allow-listed", sandbox_mode = "danger-full-access"). Created by `agentbox update` from `codex-config.toml.tmpl`.
- `.devcontainer/sync-codex-settings.sh` — Script to sync codex config into volume-mounted `~/.codex/`. Created by `agentbox update` from `sync-codex-settings.sh.tmpl`.

### Files to Modify (via agentbox update)

- `.devcontainer/Dockerfile` — Agentbox-managed stage changes: `RUN npm install -g @anthropic-ai/claude-code` becomes `RUN npm install -g @anthropic-ai/claude-code @openai/codex`, comment changes from `# --- Claude Code ---` to `# --- Coding tools ---`. Custom stage preserved verbatim.
- `.devcontainer/devcontainer.json` — Adds `"openai.chatgpt"` extension, `agentbox-codex-config` volume mount, `containerEnv` with `OPENAI_API_KEY`, updates `postStartCommand` to chown `.codex` dir and run `sync-codex-settings.sh`.
- `.devcontainer/init-firewall.sh` — Adds `ipset=/api.openai.com/allowed_ips` and `ipset=/auth.openai.com/allowed_ips` to dnsmasq config.
- `.devcontainer/dynamic-domains.conf` — Adds `api.openai.com` and `auth.openai.com` entries.
- `.devcontainer/README.md` — Adds OpenAI domains to the dynamic domains table.
- `.agentbox.yml` — Updated timestamp (cosmetic).
