Rename the entire project from "agentbox" to "agentbox" to decouple the tool from Claude Code branding and support other AI coding agents (Codex, OpenCode, etc.).

## Scope

**Code changes:**
- Go module path: `github.com/bjro/agentbox` → `github.com/bjro/agentbox`
- All Go import paths across every `.go` file
- CLI binary name and Cobra command: `agentbox` → `agentbox`
- Config file: `.agentbox.yml` → `.agentbox.yml`
- Config struct field: `agentbox_version` → `agentbox_version`
- Version ldflags path
- GoReleaser config (binary name, homepage, description)
- GitHub Actions workflows (any repo references)
- All templates (README, Dockerfile, devcontainer.json, scripts)
- CLAUDE.md and all rule files in `.claude/rules/`

**Bean migration:**
- Rename all 34 bean files: `agentbox-xxxx` → `agentbox-xxxx` (keep suffix)
- Update all cross-references inside bean files (parent, mentions)
- Update bean references in CLAUDE.md

**Infrastructure:**
- Update git remote to `github.com/BjRo/agentbox`
- Update Homebrew tap references

## Approach

Single big-bang PR. No backward compatibility needed (pre-1.0).

## Implementation Plan

### Approach

Mechanical find-and-replace executed in a strict dependency order. Go module path changes first (since all Go files depend on it), then config/struct changes, then templates, then documentation, then beans, then infrastructure. Each step is independently verifiable. The rename is purely textual -- no logic changes, no new features, no refactoring beyond the rename itself.

The key insight is that "agentbox" appears in four distinct roles across the codebase, each requiring different replacement handling:

1. **Go module path** (`github.com/bjro/agentbox`) -- becomes `github.com/bjro/agentbox`
2. **CLI binary/command name** (`agentbox`) -- becomes `agentbox`
3. **Config file/field names** (`.agentbox.yml`, `agentbox_version`, `CcboxVersion`) -- becomes `.agentbox.yml`, `agentbox_version`, `AgentboxVersion`
4. **Bean ID prefix** (`agentbox-xxxx`) -- becomes `agentbox-xxxx`

Docker volume names (`agentbox-bash-history`, `agentbox-claude-config`) and dnsmasq config path (`agentbox-dynamic.conf`) also rename to `agentbox-*`.

### Ordering Rationale

Steps 1-2 must come first (Go module path is the foundation). Steps 3-12 update Go source and tests. Steps 13-18 update non-Go files. Steps 19-21 rename config/bean files (done last because bean file renames are the most disruptive to git history and should be a clean atomic batch). Step 22 is infrastructure. Step 23 is verification.

However, for implementation efficiency, all text replacements can be done in a single pass per file since there are no ordering dependencies between the replacements within a file.

### Edge Cases

- **Bean body contains `agentbox` both as tool name and as bean ID prefix**: The simple `sed 's/agentbox/agentbox/g'` handles both cases correctly since `agentbox-xxxx` contains `agentbox` as a prefix.
- **`.claude/projects/-Users-bjro-Sources-agentbox/` directory**: This is a Claude Code memory directory path. The directory name reflects a filesystem path on the user's machine. This should **NOT** be renamed -- it is an external artifact of the user's local filesystem path, not a project reference. Leave it as-is.
- **`go.sum` file**: Will be automatically regenerated by `go mod tidy` after the module path change. No manual editing needed.
- **`DynamicDomains` field in stack.go**: These are actual DNS domain names (e.g., `proxy.golang.org`), not agentbox references. No changes needed.
- **`anthropic.claude-code` in devcontainer.json.tmpl**: This is a VS Code extension ID, not a agentbox reference. No change needed.

### Testing Strategy

- **Build verification**: `go build ./...` after all changes confirms import path consistency
- **Unit tests**: `go test ./...` exercises all renamed constants, struct fields, and template assertions
- **Integration tests**: `go test -tags integration ./...` exercises the full CLI pipeline with renamed config file
- **Lint**: `golangci-lint run ./...` catches unused imports or other issues
- **Manual spot-checks**: Search for any remaining `agentbox` (excluding `.git/` and `.claude/projects/`) to catch missed occurrences
- **No new tests needed**: This is a pure rename -- existing tests validate the rename is complete and correct when they pass with updated assertions
